## ğŸ”‘ Key Terms for This Module

| Term | Definition |
|------|------------|
| **Test Case** | A set of conditions to verify a specific feature works |
| **Positive Test** | Tests that verify expected behavior (happy path) |
| **Negative Test** | Tests with invalid inputs to verify error handling |
| **Edge Case** | Tests at the boundaries of acceptable input |
| **Boundary Value** | Testing at the limits (min, max, just over/under) |
| **Gherkin** | BDD syntax (Given-When-Then format) |
| **Test Scenario** | High-level description of what to test |
| **Test Suite** | Collection of related test cases |

## ğŸ†˜ Tips for QA Professionals

### Maximizing AI for Testing

1. **Always provide requirements** - AI can't read your mind
2. **Specify testing types** - Functional, security, performance, etc.
3. **Request specific counts** - "10 test cases" not "some test cases"
4. **Ask for coverage gaps** - "What did I miss?"
5. **Iterate** - Refine and add based on AI output
